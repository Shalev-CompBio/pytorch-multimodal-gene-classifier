# Data Requirements and Processing

## Overview
This project requires three distinct input data modalities: Phylogenetic Profiling (NPP), Protein Language Model embeddings (ESM2), and Protein-Protein Interaction networks (PPI). Due to size limitations and licensing, large data files are not distributed directly with this repository.

## Required Files
The following files must be present in the `input/` directory to run the full classification pipeline:

* **NPP_canonical_symbols_260626.csv** (~240MB) — provided by the Tabach Lab (not publicly available)
* **esm2_650M_embeddings.npz** (~91MB) — generated by running `HPC_run/esm2/esm2_inference.py`
* **ppi_node2vec_128.npz** (~12MB) — generated by running the PPI pipeline (`ppi/scripts/`)
* **sequences_for_esm2.fasta** (~11MB) — derived from UniProt Swiss-Prot human canonical proteome (https://www.uniprot.org)
* **gene_classification_SAMPLE.csv** — included in the repository under `input/` as a synthetic demonstration file. Users must supply their own labeled dataset following the format demonstrated in the sample file, or contact the authors to request access.
* **training_set_npp_esm2_ppi.npz** (~5.5MB) — generated by running `scripts/align_npp_esm2.py` then `scripts/append_ppi_embeddings.py`
* **unlabeled_set_npp_esm2_ppi.npz** (~249MB) — generated by the same two scripts

## Processing Order
To go from raw sources to training-ready data, follow these steps in order:

1. Download UniProt Swiss-Prot human canonical proteome:
   - Query: `organism_id:9606 AND reviewed:true` (Human, Swiss-Prot reviewed only)
   - Download format: FASTA (canonical sequence)
   - Save as: `sequences_for_esm2.fasta` in `input/`
2. Run ESM2 inference on the cluster → `esm2_650M_embeddings.npz`
3. Download STRING v12.0 protein network files for human (taxon 9606) from https://string-db.org → place in `ppi/raw/`. Exact files required: `9606.protein.links.v12.0.txt.gz`, `9606.protein.aliases.v12.0.txt.gz`, and `9606.protein.info.v12.0.txt.gz`.
4. Run PPI pipeline (`ppi/scripts/01` through `04`) → `ppi_node2vec_128.npz`
5. Run `scripts/align_npp_esm2.py` then `scripts/append_ppi_embeddings.py` → final `.npz` datasets

## Data Formats

### Classification CSV (`gene_classification_SAMPLE.csv`)
Required column structure:
* `gene` (string): Gene symbol
* `module_id` (integer): ID of the IRD module (0-16)
* `module_qc_label` (string): Label string (e.g., LARGE_MODULE_0, MODULE_3)
* `stability_score` (float): Score between 0.0 and 1.0
* `classification` (string): 'core', 'peripheral', or 'unstable'

### `.npz` File Arrays and Shapes

**esm2_650M_embeddings.npz**
* `gene_names`: shape=(20007,), dtype=<U25
* `embeddings`: shape=(20007, 1280), dtype=float32

**ppi_node2vec_128.npz**
* `gene_names`: shape=(20007,), dtype=<U25
* `embeddings`: shape=(20007, 129), dtype=float32

**training_set.npz**
* `gene_names`: shape=(434,), dtype=<U25
* `npp`: shape=(434, 1905), dtype=float32
* `esm2`: shape=(434, 1280), dtype=float32
* `labels`: shape=(434,), dtype=int32

**unlabeled_set.npz**
* `gene_names`: shape=(19573,), dtype=<U25
* `npp`: shape=(19573, 1905), dtype=float32
* `esm2`: shape=(19573, 1280), dtype=float32

**training_set_npp_esm2_ppi.npz**
* `gene_names`: shape=(434,), dtype=<U25
* `npp`: shape=(434, 1905), dtype=float32
* `esm2`: shape=(434, 1280), dtype=float32
* `labels`: shape=(434,), dtype=int32
* `ppi`: shape=(434, 129), dtype=float32

**unlabeled_set_npp_esm2_ppi.npz**
* `gene_names`: shape=(19573,), dtype=<U25
* `npp`: shape=(19573, 1905), dtype=float32
* `esm2`: shape=(19573, 1280), dtype=float32
* `ppi`: shape=(19573, 129), dtype=float32

## Note on NPP Data
The NPP (Normalized Phylogenetic Profiling) matrix is proprietary lab data and is not currently available for public download. Readers or researchers interested in reproducing these specific results using the NPP features should contact the original authors to request data access.
